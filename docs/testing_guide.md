# 小范围测试指南

## 测试目标

找 3-5 人试用 3 天，收集真实反馈。

## 测试人员要求

- 使用 Android 手机（当前只构建了 APK）
- 每天有阅读新闻的习惯
- 愿意提供简单反馈

## 测试流程

### 第一步：分发 APK

1. 从 GitHub Releases 下载最新 APK
2. 通过微信/钉钉发送给测试人员
3. 让他们安装并打开

### 第二步：收集反馈

请测试人员回答以下问题：

#### 基础体验
1. 第一眼看到 app 的感觉？
2. 今天的内容推荐符合你的兴趣吗？
3. 阅读体验流畅吗？有没有卡顿？

#### 功能测试
4. 点击"朗读"按钮，能正常播放吗？
5. 切换主题（深色/浅色）正常吗？
6. 收藏文章后，在"收藏"页面能找到吗？
7. 断网时显示的页面合理吗？

#### 产品感受
8. 每天 10 条内容够吗？太多还是太少？
9. 有什么功能是你期望有但没有的？
10. 你会推荐给朋友吗？为什么？

### 第三步：记录问题

让测试人员记录遇到的任何问题：

```
问题描述：[他们看到了什么]
操作步骤：[他们做了什么]
期望结果：[他们期望发生什么]
实际结果：[实际发生了什么]
```

### 第四步：总结反馈

测试结束后，整理所有反馈：

| 问题 | 出现次数 | 优先级 |
|---|---|---|
| xxx | 3 | 高 |
| xxx | 1 | 低 |

## 注意事项

- 不要给测试人员太多指导，让他们自然使用
- 记录他们"卡住"的地方
- 关注他们"没想到"的功能
- 收集他们对产品定位的理解

## 测试时间线

```
Day 1: 安装 + 第一印象
Day 2: 日常使用
Day 3: 收集反馈
```
